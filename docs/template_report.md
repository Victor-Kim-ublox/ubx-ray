# templates/report.html

## Overview
The **analysis result report page** for a single UBX file. Polls conversion status in real time and, once complete, displays epoch statistics, graph charts, and links to the map viewer and KMZ download.

---

## URL
`GET /report/{rid}`

---

## Jinja2 Template Variables

| Variable | Type | Description |
|---|---|---|
| `r` | dict | DB results row (id, filename, uploaded_at, epoch_total, epoch_missing, crc_errors, kmz_path, opts_json, status, error) |
| `graph_data` | dict or None | Graph JSON generated by ubx2kmz (`_graph.json`) |

---

## Screen Layout

### Top
- Page title: "Analysis Result"
- Back link: `← Home`

### Status Card
Conversion status badge:
- `queued` / `running` → yellow "Processing..." + spinner + auto-polling (every 2 seconds)
- `done` → green "Done ✓"
- `error` → red "Error: {message}"

### File Info Card

| Field | Description |
|---|---|
| Filename | Original filename |
| Uploaded | Upload timestamp (UTC) |
| Epoch Total | NAV-PVT frame count (from fast scan; updated after conversion) |
| Epoch Missing | Missing epoch count (updated after conversion completes) |
| CRC Errors | Checksum error count |
| Options | Conversion options used (parsed from opts_json) |

### Conversion Options Display
Parses `opts_json` and shows only active options as badges:
```javascript
const opts = JSON.parse(r.opts_json);
// hz, nav2, alt_abs, ck, mapm → display only enabled options as badges
```

### Action Buttons (active only when `done`)
- `📍 View on Map` → `/map/{rid}`
- `⬇ Download KMZ` → `/download?path={kmz_path}`

### Graph Section (when graph_data is available)
Chart.js charts (all share an X axis via `syncCharts`):
1. **Position Accuracy** — 2D vs 3D (m)
2. **Fix Type** — 0–5 step line
3. **CN0 Density** — top-5 average line + per-SV scatter
4. **Jamming & Spoofing Status** — rendered only when `graph_data.sec_labels` is non-empty. Two stepped lines: `jam_state` (0=Unknown, 1=OK, 2=Warning) and `spf_state` (0=Unknown, 1=OK, 2=Indicated, 3=Affirmed).

Chart interaction: zoom/pan (chartjs-plugin-zoom + hammerjs), "Hide Chart" per section, and "Reset All" to unzoom every chart at once.

## Auto-Polling Logic

```javascript
// Poll every 2 seconds if status is queued or running
if (r.status in ['queued', 'running']) {
  setTimeout(() => {
    fetch(`/api/status/${rid}`)
      .then(res => res.json())
      .then(data => {
        if (data.status === 'done') location.reload();
        else scheduleNextPoll();
      });
  }, 2000);
}
```

---

## Epoch Missing Display Logic

- `epoch_missing > 0` → warning badge (yellow) showing count and percentage
- `epoch_missing == 0` → normal badge (green)
- Before conversion (`status != done`) → shows "Calculating..."
