## Development Commands

```bash
# Activate virtual environment (Windows)
.venv\Scripts\activate

# Run development server with auto-reload
uvicorn app:app --reload

# Run on custom host/port
uvicorn app:app --host 0.0.0.0 --port 8000 --reload
```

No build step required — pure Python application. Dependencies: FastAPI, Jinja2, uvicorn, and optionally pyubx2.

## Architecture Overview

Three main Python modules:

**`app.py`** — FastAPI backend. Handles all HTTP routes, SQLite DB (WAL mode), user isolation via anonymous UUID cookies, file upload validation (UBX magic bytes), rate limiting (sliding window deque), and a background cleanup daemon (TTL/quota/per-user limits). Concurrency controlled via `asyncio.Semaphore` + `ThreadPoolExecutor`. Admin override via `UBXRAY_ADMIN_TOKEN` bearer token.

**`ubx2kmz.py`** — Core UBX binary parser and KMZ generator. Uses `mmap`/`memoryview` for zero-copy parsing of NAV-PVT frames. Produces KMZ (OpenLayers-renderable) and JSON graph data (accuracy, speed, altitude, CN0). Supports NAV2-PVT (gen10+ devices), AID-MAPM (external heading), Hz downsampling, checksum verification.

**`nmea_comparison.py`** — NMEA GGA parser for position error analysis. Computes Haversine distances and CEP50/CEP95 statistics.

## Data Flow

```
Upload → quick_ubx_summary() [fast mmap frame count]
       → DB INSERT (status=queued)
       → enqueue_convert() → subprocess → ubx2kmz
         → KMZ + graph JSON written to outputs/{rid}/
         → DB UPDATE (done/error)
       → /report/{rid} polls /api/status/{rid} until done
```

Multi-file (4-way compare) follows the same per-file pipeline, then `/compare4/report/{r1}/{r2}/{r3}/{r4}` renders split or overlay map views.

NMEA comparison (`/analyze_nmea`) runs synchronously in threadpool — results are not persisted to DB.

## Admin CLI

`admin_cli.py` (doc: `docs/admin_cli.md`) is a standalone tool for listing and
deleting stored uploads directly from the server shell. Dry-run by default;
`--yes` to commit. Example: `python admin_cli.py purge --older-than 7 --yes`.

## Key Configuration

| Env Var | Default | Purpose |
|---------|---------|---------|
| `UBXRAY_MAX_UPLOAD_MB` | 300 | Max upload file size |
| `UBXRAY_ADMIN_TOKEN` | (none) | Bearer token to bypass ownership checks |

Hardcoded limits in `app.py`: 10 GB total disk quota, 10 results per user, 7-day TTL, cleanup every 3600s, `MAX_CONVERT` = CPU cores ÷ 2.

## File Layout

```
data/ubxray.sqlite3     ← auto-created SQLite DB
uploads/                ← uploaded UBX files + graph JSON
outputs/{rid}/          ← result.kmz per conversion
templates/              ← 9 Jinja2 HTML templates
```

## Frontend

No build toolchain. Templates use CDN-loaded OpenLayers 9.1.0 (maps) and Chart.js 4.4.0 (graphs). Map playback and 4-way split/overlay logic lives in the templates.
